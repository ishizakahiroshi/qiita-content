ブラウザ版は `offline-md-editor-viewer.html` を開くだけで動きます。Windows向けにはTauriで作ったポータブル `.exe` も配布しています。

## 何を作ったか

`offline-md-editor-viewer` は、ローカルのMarkdownファイルを開いて編集し、ライブプレビューし、そのまま元ファイルへ保存するツールです。

基本の流れはこれだけです。

```mermaid
flowchart LR
    A[フォルダを開く] --> B[ファイルを選ぶ]
    B --> C[左で編集]
    C --> D[右でプレビュー]
    D --> E[上書き保存]
    C -.即時反映.-> D
```

主な機能:

- Markdown編集
- ライブプレビュー
- 左右スクロール同期
- フォルダツリー表示
- ファイル/フォルダの作成、名前変更、削除
- ツリー内ドラッグ&ドロップ移動
- 検索・置換
- ダーク/ライトテーマ
- UI 13言語対応
- Windowsポータブル版

Markdown以外のテキストファイルも開けます。`.json`、`.yml`、`.env`、`.txt` などはプレビューなしでソース側だけ編集できます。

## 方針

このツールでは、次の点を重視しました。

- アカウント不要
- 同期不要
- ファイルアップロードなし
- CDNなし
- 外部通信なし
- ブラウザ版はビルド不要
- Windows版はインストール不要

高機能なノートアプリというより、「手元のMarkdownを安全に開いて編集する小さな道具」を目指しています。

## ブラウザ版: 単一HTML

ブラウザ版は、リリース時に1つのHTMLへまとめています。

配布ファイル:

```text
offline-md-editor-viewer.html
```

このHTMLの中に、実行に必要なライブラリやアイコンを同梱しています。

- `marked`
- `DOMPurify`
- `encoding-japanese`
- `Feather Icons`

そのため、アプリ起動時にCDNへアクセスしません。ダウンロードしたHTMLをChromeで開けば、そのままオフラインで使えます。

リポジトリ内のソース版HTMLは `apps/browser/lib/` からライブラリを読み込む構成ですが、リリース版ではビルドスクリプトでインライン化しています。

## Markdownレンダリング

レンダリングは大まかに以下の流れです。

```text
Markdown source
→ marked.parse()
→ DOMPurify.sanitize()
→ preview.innerHTML
```

Markdownのパースには `marked`、HTMLのサニタイズには `DOMPurify` を使っています。

MarkdownエディタではプレビューにHTMLを入れる必要があるので、サニタイズを挟む構成にしています。

## 外部通信をしない

このツールの特徴として、外部通信をしないことを明確にしています。

READMEにも「Security / Privacy」節を作りました。

実装・配布上のポイント:

- ライブラリはCDNではなく同梱
- リリース版HTMLは `connect-src 'none'` のCSPを設定
- ファイル内容を外部サーバーへ送信しない
- Markdown内のリモート画像URLはCSPでブロック
- 外部リンクは設定を有効にした場合だけ開く

特に `connect-src 'none'` は、アプリのスクリプトから外部通信を行わないことをブラウザ側でも制限するためのものです。

## ファイルアクセス

ブラウザ版では、ChromeのFile System Access APIを使います。

できること:

- ファイルを開く
- フォルダを開く
- 元ファイルへ上書き保存
- フォルダツリー表示
- 対応ブラウザではファイル/フォルダの作成・変更・削除

ブラウザのサンドボックスがあるため、ユーザーが明示的に選んだファイルやフォルダだけを扱います。

一方で、DesktopやDocumentsなどの特殊フォルダはChrome側で保護されることがあります。その場合は通常のサブフォルダを作って開く運用にしています。

## Windows版: Tauri

Windowsデスクトップ版はTauriで作っています。

配布ファイル:

```text
offline-md-editor-viewer.exe
offline-md-editor-viewer-desktop-vX.X.X-win-x64-portable.zip
```

インストーラーは作らず、ポータブル `.exe` として配布しています。

デスクトップ版でも、フロントエンドはブラウザ版と同じHTMLを使います。TauriのWebViewに同じHTMLを読み込ませる構成です。

デスクトップ版で追加しやすかったこと:

- OSのファイルダイアログ
- 前回開いたフォルダパスの復元
- 単体ファイルを開いたときの親フォルダツリー表示
- Explorerの「プログラムから開く」連携
- フォルダドラッグ&ドロップ

Windows版ではWebView2のユーザーデータをexeと同じ階層の `offline-md-editor-viewer-userdata/` に置くようにして、exeと一緒に持ち運びやすくしています。

## リリース構成

`v0.1.0` では、リリースアセットを4種類にしました。

```text
offline-md-editor-viewer.html
offline-md-editor-viewer.exe
offline-md-editor-viewer-browser-vX.X.X.zip
offline-md-editor-viewer-desktop-vX.X.X-win-x64-portable.zip
```

さらに、各リリースに `SHA256SUMS.txt` を添付しています。

CIでは以下を確認しています。

- ブラウザZIP内のHTMLがスタンドアロンHTMLと一致する
- デスクトップZIP内のexeがスタンドアロンexeと一致する
- SHA-256チェックサムを生成する
- リリースノートにチェックサムを載せる

配布物が複数になると、単体ファイルとZIP内ファイルの一致を確認したくなるので、ここは自動化しました。

## 作ってみてよかったところ

単一HTML配布は、使う側の説明がとても短くなります。

```text
ダウンロードして開いてください
```

で済みます。

また、ブラウザ版とTauri版で同じHTMLを使うことで、フロントエンドの挙動差を減らせました。ブラウザで動くものを、そのままWindowsアプリにも載せられるのは便利でした。

一方で、ブラウザのファイルアクセスはユーザー許可や特殊フォルダ制限があるため、そこはREADMEで丁寧に説明する必要がありました。
